# Team-C Real Test Brief

## Goal

Run a real `team-c` workflow against the current `team-c` and `team-codex` implementation.

## Task

Evaluate whether the current implementation is ready for real use in this workspace.

Each teammate focuses on a distinct angle:

- `architect`: runtime design, protocol fidelity, execution mapping
- `critic`: misleading claims, hidden semantic gaps, where it still diverges from Claude agent teams
- `qa`: usability, installability, operational risk, verification gaps

## Constraints

- Read-only analysis only
- No code changes by teammates
- Each teammate writes conclusions for the lead to integrate

## Expected output

- one per-role conclusion file
- one lead synthesis
