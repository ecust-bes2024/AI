# 海报“个性化”视觉语言迁移研究

**日期**: 2026-03-29
**目标**: 研究如何把样张中那种明显“有个性”的海报风格，稳定迁移到当前本地 `poster` 生成器里。

**研究对象**:

- 样张 1 / 样张 2（手账 / 学习卡片 / scrapbooking 风格）
- 当前本地生成器：
  - [poster_generator.py](/Users/jerry_hu/.codex/skills/poster/poster_generator.py)

---

## 一、先说结论

这套“有个性”的感觉，不是来自某一个元素，而是来自一套稳定的视觉语言组合：

1. **纸张感**
2. **贴纸感**
3. **手写感**
4. **轻微不对称**
5. **拟物小装饰**
6. **模块卡片的边框感**

当前生成器已经有第 1 条的一半：

- 米白纸张底
- 柔和背景
- 卡片系统

但还缺其余 5 条的大部分，因此现在生成出来更像：

- “好看的信息海报”

而不是：

- “有作者风格的学习海报”

---

## 二、样张到底靠什么成立

### 1. 不是普通信息图，而是“学习笔记板”

样张不是在模仿商业 dashboard，而是在模仿：

- 手账
- 学习卡片
- 课程讲义
- 贴纸式知识摘要

这决定了它的视觉目标不是“严谨整齐”，而是“有人的整理痕迹”。

### 2. 标题的个性来自字体角色分裂

标题明显用了两种角色不同的字形：

- 主标题：手写/书写感，情绪很强
- 副标题或标签：更窄、更规整、更像贴纸标签

当前生成器的问题是：

- 整张海报基本还是同一类系统字体
- 只有字号和粗细变化，没有“人格变化”

### 3. 视觉焦点靠“贴纸式层叠”而不是靠纯排版

样张里很多信息块像被贴上去：

- 顶部彩色角标
- 版本徽章
- 小胶带
- 圆角标签
- 卡片外框
- 手写批注

这些元素让信息不只是“放上去”，而是“被贴上去”。

这会带来非常强的作者感。

### 4. 不对称是受控的，不是杂乱

个性来自轻微失衡：

- 胶带不在几何中心
- 注释斜着放
- 角标不是完全对称
- 局部有“手工贴上去”的偏移

但整体仍然有大结构：

- 上中下分段
- 每一段还是独立模块

所以这不是随意乱排，而是：

- **宏观有秩序**
- **微观有人味**

### 5. 模块卡片有“硬边框 + 圆角 + 厚重感”

样张里的信息卡不是轻飘飘的玻璃卡片，而是：

- 较粗边框
- 强圆角
- 明显投影
- 黑色/深色代码卡嵌在暖色背景卡里

这类“卡中卡”结构很有辨识度。

### 6. 细碎纹理在放大“纸感”

底部纸张不是纯色：

- 有淡噪点
- 有纸面纹理
- 有轻微旧纸色差

当前生成器虽然也有渐变纸色，但偏“干净”，不够“材质化”。

---

## 三、当前 poster 生成器缺什么

当前生成器的强项：

- 色板已经不差
- 纸感背景已经有雏形
- 卡片化信息组织很稳
- 长文结构化生成能力强

当前生成器的主要短板，不在颜色，而在这些地方：

### 1. 缺“字体角色系统”

从代码上看，当前主字体基本还是系统 sans-serif：  
[poster_generator.py](/Users/jerry_hu/.codex/skills/poster/poster_generator.py)

这意味着：

- 标题、标签、正文只是层级不同
- 不是“人格不同”

### 2. 缺“装饰组件系统”

现在的组件主要是：

- hero
- pills
- summary cards
- detail cards
- metric cards

这些都偏规整。缺少：

- tape
- sticker
- angled note
- doodle arrow
- corner ribbon
- hand annotation

### 3. 缺“轻微偏移和旋转语法”

当前布局基本是严格网格和严格居中：

- 4 列 grid
- 2 列 grid
- 对齐整洁

这会让成品“稳定”，但也让风格偏保守。

### 4. 缺“卡中卡”的戏剧性层次

样张里的代码块不是普通 `<code>`，而是：

- 深色独立卡片
- 有内边距
- 有标题条/小圆点

当前生成器代码块处理太弱，不足以成为视觉焦点。

### 5. 缺“手工批注”

样张里的 “Important!”、箭头、小提示，是人格化最强的一层。

这类元素不是内容主体，但它们决定海报有没有“人味”。

## 四、哪些可以稳定迁移

可以迁移，而且应当迁移的是“风格语法”，不是这张海报的表面样子。

### 1. 可迁移：纸张主题层

把当前主题系统从“配色变量”升级为“材质变量”：

- 纸色
- 纸纹理密度
- 噪点强度
- 阴影风格
- 边框粗细

这部分适合稳定参数化。

### 2. 可迁移：装饰组件层

做成一组可控组件，而不是每次手写 CSS：

- corner ribbon
- tape strip
- sticker badge
- hand note
- marker underline
- doodle arrow

这部分一旦组件化，就能稳定复用。

### 3. 可迁移：布局语法层

不要只用“固定 grid”，而是增加几种版式 archetype：

- `scrapbook-note`
- `study-card`
- `cheatsheet-board`

每种 archetype 规定：

- 标题样式
- 区块节奏
- 装饰使用比例
- 代码块呈现方式
- 批注位置策略

### 4. 可迁移：字体角色层

不必强依赖某一款商业字体，但要引入“字体职责”：

- title font
- label font
- body font
- annotation font
- code font

哪怕先用可访问字体近似，也比现在只靠系统 sans-serif 强很多。

### 5. 可迁移：内容映射策略

并不是所有 poster 都适合这种风格。

更适合：

- 学习指南
- 知识卡片
- cheatsheet
- 概念解释
- 课程海报

不太适合：

- 严肃架构图
- 密集表格
- 合规/财务报告
- 纯 executive summary

所以应当把这种风格做成：

- **一种主题变体**

而不是默认主题。

---

## 五、哪些不要照抄

### 1. 不要照抄“Spring Boot Learning Guide”这张的具体样子

不该复制：

- 具体排版
- 具体颜色组合
- 具体文案位置
- 具体贴纸摆放

那会变成仿品，而不是风格迁移。

### 2. 不要把所有 poster 都改成这种风格

这类风格强度高：

- 适合教学型内容
- 不适合一切内容

### 3. 不要只加贴纸，不改版式

如果只是往现有 poster 上加一点胶带或噪点：

- 风格不会成立
- 反而会显得廉价

真正要迁移的是整套语法：

- 字体角色
- 组件角色
- 版式 archetype
- 装饰密度

---

## 六、建议的迁移路线

### Phase 1: 定义风格变量

先不动大逻辑，只补一层风格 token：

- `paper_texture`
- `border_weight`
- `card_rounding`
- `shadow_style`
- `title_font_role`
- `annotation_font_role`
- `decor_density`

### Phase 2: 增加组件库

最少增加这些组件：

- `tape`
- `sticker`
- `annotation`
- `corner-ribbon`
- `code-panel-retro`

### Phase 3: 增加新主题 archetype

建议新加一个明确主题，而不是把现有 `research/technical/project` 改坏：

- `study-notes`

它专门服务这类“有个性的学习海报”。

### Phase 4: 内容映射规则

定义什么内容会映射到：

- title
- badges
- note card
- code panel
- callout annotation

也就是让风格稳定，而不是靠人手工拼。

---

## 七、最关键的判断

如果问“这套风格能不能稳定迁移”：

- `能`

如果问“该不该直接在当前 poster 上打补丁”：

- `不该`

最合理的做法是：

- 把它做成一个新的风格主题
- 定义组件和版式语法
- 只让适合的内容使用它

---

## 八、一句话结论

当前 poster 生成器已经有“纸感”和“卡片感”，但还没有“人格化视觉语法”。  
要稳定迁移这种个性，不是去抄一张图，而是要新增：

- 字体角色系统
- 装饰组件系统
- 非对称但受控的版式 archetype
- 教学型内容专用主题
