## ⚠️ 重要注意事项

### 1. 代理问题（必读！）

**你的系统配置了代理（localhost:61693），会导致飞书 API 请求失败！**

**解决方案**：所有命令前必须加上代理禁用环境变量：

```bash
HTTP_PROXY="" HTTPS_PROXY="" http_proxy="" https_proxy="" python3 lark_cli.py [命令]
```

**为什么需要禁用代理？**
- 系统代理会拦截飞书 API 请求
- 导致 SSL 证书验证失败（OSStatus -26276）
- 或返回 401/503 错误

### 2. Cookie 过期

Cookie 会定期过期（通常几天到几周），过期后需要重新提取。

**症状**：命令返回 401 未授权错误

**解决方案**：
```bash
cd /Users/jerry_hu/AI/reverse-engineering
source .venv/bin/activate
python3 extract_cookie.py
```

### 3. 虚拟环境

项目使用 Python 虚拟环境，每次使用前需要激活：

```bash
cd /Users/jerry_hu/AI/reverse-engineering
source .venv/bin/activate
```

## 常用命令

### 搜索用户或群组
```bash
HTTP_PROXY="" HTTPS_PROXY="" python3 lark_cli.py search "关键词"
```

**示例**：
```bash
# 搜索飞书助手
HTTP_PROXY="" HTTPS_PROXY="" python3 lark_cli.py search "飞书助手"
```

**输出示例**：
```
[group] 7134941989665112067  胡志胜的飞书助手
```

### 发送消息

#### 方式 1：按用户名发送（自动搜索）
```bash
HTTP_PROXY="" HTTPS_PROXY="" python3 lark_cli.py send --to "用户名" --msg "消息内容"
```

#### 方式 2：使用 chat_id 发送（推荐）
```bash
HTTP_PROXY="" HTTPS_PROXY="" python3 lark_cli.py send --chat-id "chat_id" --msg "消息内容"
```

**示例**：
```bash
# 给飞书助手发消息
HTTP_PROXY="" HTTPS_PROXY="" python3 lark_cli.py send \
  --chat-id "7134941989665112067" \
  --msg "测试消息"
```

#### 方式 3：给自己发消息
```bash
HTTP_PROXY="" HTTPS_PROXY="" python3 lark_cli.py notify "提醒内容"
```

### 实时监听消息

#### 人类可读格式
```bash
HTTP_PROXY="" HTTPS_PROXY="" python3 lark_cli.py watch
```

#### JSON 格式（适合程序处理）
```bash
HTTP_PROXY="" HTTPS_PROXY="" python3 lark_cli.py watch --json
```

**JSON 输出示例**：

## 故障排查

### 问题 1：SSL 证书错误
```
SSLError(SSLCertVerificationError('OSStatus -26276'))
```

**原因**：系统代理干扰

**解决**：确保命令前加了 `HTTP_PROXY="" HTTPS_PROXY=""`

### 问题 2：401 未授权
```
RuntimeError: Authentication failed
```

**原因**：Cookie 过期

**解决**：重新提取 Cookie
```bash
cd /Users/jerry_hu/AI/reverse-engineering
source .venv/bin/activate
python3 extract_cookie.py
```

### 问题 3：找不到用户
```
RuntimeError: No user found for 'xxx'
```

**原因**：
- 用户名拼写错误
- 或者目标是群组（群组不能用 `--to` 参数）

**解决**：
- 先用 `search` 命令搜索，获取 chat_id
- 然后用 `--chat-id` 参数发送

### 问题 4：虚拟环境未激活
```
ModuleNotFoundError: No module named 'requests'
```

**解决**：激活虚拟环境
```bash
cd /Users/jerry_hu/AI/reverse-engineering
source .venv/bin/activate
```

## 技术细节

### 认证方式
- 使用 Cookie 认证（非官方 API）
- Cookie 存储在：`~/.config/lark-cli/config.json`
- 自动从 Chrome 浏览器提取

### API 端点
- 内部 API：`https://internal-api-lark-api.feishu.cn/im/gateway/`
- WebSocket：`wss://msg-frontier.feishu.cn/ws/v2`

### 协议
- 使用 Protobuf 编码
- 三层嵌套：Frame → Packet → 业务消息

### 项目结构
```
reverse-engineering/
├── .venv/              # Python 虚拟环境
├── lark_cli.py         # CLI 入口
├── lark_api.py         # API 客户端
├── lark_daemon.py      # WebSocket 实时监听
├── lark_proto.py       # Protobuf 编解码
├── lark_utils.py       # 工具函数
├── proto_pb2.py        # Protobuf 定义（自动生成）
├── extract_cookie.py   # Cookie 提取工具
├── requirements.txt    # Python 依赖
├── README.md           # 英文文档
└── 使用说明.md         # 本文档
```
