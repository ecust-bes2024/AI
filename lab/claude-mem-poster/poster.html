项目概述

claude-mem 是 Claude Code 的持久化内存压缩系统，自动捕获工具调用、文件变更、决策，通过 AI 压缩成可搜索记忆，并在未来会话中自动注入相关上下文。

💎 核心价值
解决的核心问题
Claude Code 会话结束后上下文丢失，导致每次重新开始都需要重新建立上下文。
关键特性
  • 自动化捕获：5 个生命周期 hooks 自动记录所有操作
  • AI 压缩：使用 Claude agent-sdk 压缩，v10.6.1 实现 53% token 节省
  • System Prompt 注入：v10.6.0+ 通过 before_prompt_build hook 注入，不写 MEMORY.md
  • 60 秒缓存：每项目缓存，减少重复注入开销
  • 选择性注入：syncMemoryFileExclude 排除特定 agent
  • Timeline Reports：生成项目演进叙事报告
✨ 值得学习
  • System Prompt 注入架构：通过 hook + appendSystemContext，不写 MEMORY.md，保持用户控制
  • Observations 数据结构：捕获工具调用、文件变更、决策、时间戳，AI 压缩成可搜索记忆
  • 选择性注入机制：syncMemoryFileExclude 排除噪音 agent，减少 token 消耗
  • 缓存策略：60 秒/项目平衡新鲜度和性能
  • 三层检索模式：索引 → 时间线 → 完整细节，10x token 节省
⚠️ 不建议复制
  • Bun runtime：增加运行时复杂度
  • Chroma 向量数据库：如果已有 LanceDB，重复投资
  • Worker daemon 架构：独立进程 + HTTP API，复杂度高
  • Web UI：localhost:37777 的 React UI，非核心功能
  • 自动捕获所有工具调用：可能导致高 token 消耗
