# Role Roster

This roster is adapted from the local Claude `team` skill.

Use only the roles needed for the task.

| Role | Slug | Best for |
|---|---|---|
| CEO | `ceo` | strategy, prioritization, PR/FAQ, business framing |
| CTO | `cto` | architecture, system design, technical choices |
| Critic | `critic` | pre-mortem, failure analysis, challenging assumptions |
| Product | `product` | product definition, UX goals, scope |
| UI | `ui` | visual design, layout, design system direction |
| Interaction | `interaction` | flows, user journeys, behavior design |
| Fullstack | `fullstack` | implementation, code changes, app wiring |
| QA | `qa` | test strategy, regression risk, quality gates |
| DevOps | `devops` | CI/CD, infra, deployment, observability |
| Marketing | `marketing` | positioning, messaging, growth content |
| Operations | `operations` | user ops, community, PMF, rollout process |
| Sales | `sales` | funnel, conversion, sales process |
| CFO | `finance` | pricing, cost control, unit economics |
| Research | `research` | market scan, competitor analysis, evidence gathering |

## Selection heuristics

- Strategy-heavy: `ceo`, `cto`, `critic`
- Product-heavy: `product`, `ui`, `interaction`
- Build-heavy: `cto`, `fullstack`, `qa`, `devops`
- Go-to-market-heavy: `marketing`, `operations`, `sales`, `finance`
- Discovery-heavy: `research`, `critic`, one domain expert
